# Learn & Colab - Messaging Framework

## Learn (Executives/Leaders New to Claude Code)

### Positioning Sentence
Claude Code for decision-makers: understand what it is, how it fits your org, and how to roll it out safely.

### 3 Outcomes (Executive Language)
1. **Decision-ready clarity** — Understand Claude Code well enough to green-light or de-risk it for your team
2. **Safe first steps** — A practical rollout plan with guardrails, not a reckless experiment
3. **Team alignment** — Everyone from leadership to operators on the same page

### "You'll Leave With..." (Artifacts)
- Executive briefing (1–2 pages) explaining Claude Code in business terms
- Recommended rollout plan (2–4 weeks) with approval gates
- "First use-case shortlist" ranked by ROI and risk
- Team enablement plan (who does what, when)
- Security and governance checklist

### "Not For You If..."
- You want someone to code for you (that's Colab)
- You're already shipping with Claude Code (that's Colab)
- You need a multi-month implementation engagement (that's Colab)

## Colab (Next-Level Execution)

### Positioning Sentence
Implementation partner in your repo: we ship Claude Code workflows, plugins, and agents alongside your team.

### 3 Outcomes (Delivery Language)
1. **Production code merged** — Real PRs in your repo, reviewed and deployed
2. **Hardened workflows** — CI guardrails, evals, and observability in place
3. **Clean handoff** — Documentation and training so your team can maintain it

### "We Ship..." (Deliverables)
- Claude Code repo setup with CLAUDE.md, hooks, and permissions
- Custom plugins and skills for your workflows
- CI/CD guardrails (human approval gates, test suites)
- Agent implementations on Vertex AI / ADK
- Deployment runbooks and handoff docs

### "Not For You If..."
- You're not sure what Claude Code is (start with Learn)
- You don't have a codebase or team to work with
- You want strategic advice only (Learn might be better)
